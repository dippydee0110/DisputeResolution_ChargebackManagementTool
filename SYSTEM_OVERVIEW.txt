🎯 HOW IT WORKS
═════════════════════════════════════════════════════════════════════════════════

INPUT:  Customer chat transcripts
        ↓
        evidence/CASE123/chat_CASE123.txt
        (Plain text customer-support conversations)
        ↓

PROCESSING: AI Analysis
        ↓
        Detects 8 warning signals:
        🔴 Unrecognized transaction (weight 25%)
        🔴 Refund request (weight 20%)
        🔴 Item not received (weight 20%)
        🔴 Duplicate charge (weight 18%)
        🟠 Merchant mismatch (weight 15%)
        🟠 Quality complaint (weight 15%)
        🟠 Frustration escalation (weight 15%)
        🟠 Urgent tone (weight 10%)
        ↓

OUTPUT: Risk Score (0.0-1.0) + Recommendation
        ↓
        🔴 CRITICAL (70%+) → Contact customer NOW
        🟠 HIGH (50-70%) → Priority review
        🟡 MEDIUM (30-50%) → Monitor
        🟢 LOW (0-30%) → Standard process

📞 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════════

Python not found?
→ Install from python.org
→ Add to PATH during installation

openpyxl not found?
→ Run: pip install openpyxl

No results generated?
→ Check evidence/CASEID/chat_CASEID.txt exists
→ Ensure .txt file has content

Risk scores wrong?
→ Adjust signal weights in early_dispute_detection.py
→ Add more patterns for your use case

Can't run scripts?
→ Use full path: python "c:\full\path\to\script.py"
→ Check file permissions
